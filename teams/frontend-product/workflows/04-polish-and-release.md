# Workflow 04: Polish and Release

## Goal

Take a working slice and make it trustworthy enough for real user feedback.

## Checklist

- empty states are intentional
- loading states are visible
- obvious errors are surfaced
- keyboard path works for core actions
- panel layout holds on common screen sizes
- build/test verification is documented
- one concise backlog for the next slice exists

## Exit condition

A human can use the slice, understand its edges, and give product feedback
without needing a narrated walkthrough.
