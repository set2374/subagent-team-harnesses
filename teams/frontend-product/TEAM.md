# Frontend Product Team

## Purpose

Deploy a professional three-panel workbench UI with durable chat, research, and
document surfaces without overbuilding the stack too early.

## Standard specialists

1. Frontend Architect
2. Workbench UI Designer
3. Frontend App Engineer
4. Backend Integration Engineer
5. QA and Accessibility Engineer

## Team operating model

- Architect defines the slice and ownership boundaries
- Designer sharpens interaction and visual direction
- App engineer builds the shell and client code
- Backend integration engineer exposes the minimal contract needed
- QA checks usability, regressions, and release fitness

## Persistent-team rule

Keep these agents alive across iterations unless the human says to close them.
Update their memory artifacts after each meaningful slice.

## Default working order

1. kickoff and slice framing
2. UI and API boundary alignment
3. scaffold and implement
4. validate and harden
5. capture memory and next slice

## Typical outputs

- architecture note
- API contract note
- runnable scaffold
- findings report
- next-slice backlog
