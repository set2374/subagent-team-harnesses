# Workflow 01: Kickoff and Slice

## Goal

Turn a vague frontend idea into a bounded implementation slice that multiple
specialists can work on in parallel.

## Steps

1. Write the user-facing objective in one sentence.
2. Define the smallest shippable slice.
3. Split ownership by surface:
   - architecture
   - UI direction
   - app code
   - backend contract
   - QA
4. Record:
   - must ship now
   - can fake for now
   - explicitly deferred
5. Seed each persistent agent with the same objective and a bounded write scope.

## Output

- kickoff note
- ownership map
- first deliverables per agent
